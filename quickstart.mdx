# Quick Start Guide

Get up and running with the Tattoo API in just a few minutes. This guide will walk you through making your first API request and understanding the basic concepts.

## Step 1: Get Your API Key

<Steps>
  <Step title="Sign Up">
    Create an account at [tattoo.co/developers](https://tattoo.co/developers)
  </Step>
  <Step title="Verify Email">
    Check your email and verify your account
  </Step>
  <Step title="Generate API Key">
    Navigate to your dashboard and generate your first API key
  </Step>
  <Step title="Copy & Store">
    Copy your API key and store it securely (you'll need it for all requests)
  </Step>
</Steps>

<Warning>
**Keep your API key secure!** Never expose it in client-side code or commit it to version control. Use environment variables in production.
</Warning>

## Step 2: Make Your First Request

Let's start by fetching a list of tattoo studios. Here's how to do it in different programming languages:

<CodeGroup>

```bash cURL
curl -X GET "https://api.tattoo.co/v1/studios" \
  -H "Authorization: Bearer YOUR_API_KEY" \
  -H "Content-Type: application/json"
```

```javascript JavaScript
const response = await fetch('https://api.tattoo.co/v1/studios', {
  method: 'GET',
  headers: {
    'Authorization': 'Bearer YOUR_API_KEY',
    'Content-Type': 'application/json'
  }
});

const data = await response.json();
console.log(data);
```

```python Python
import requests

headers = {
    'Authorization': 'Bearer YOUR_API_KEY',
    'Content-Type': 'application/json'
}

response = requests.get('https://api.tattoo.co/v1/studios', headers=headers)
data = response.json()
print(data)
```

```php PHP
<?php
$curl = curl_init();

curl_setopt_array($curl, array(
  CURLOPT_URL => 'https://api.tattoo.co/v1/studios',
  CURLOPT_RETURNTRANSFER => true,
  CURLOPT_HTTPHEADER => array(
    'Authorization: Bearer YOUR_API_KEY',
    'Content-Type: application/json'
  ),
));

$response = curl_exec($curl);
curl_close($curl);

$data = json_decode($response, true);
print_r($data);
?>
```

</CodeGroup>

## Step 4: Filter and Search

You can refine your results using query parameters:

<CodeGroup>

```bash Search by Location
curl -X GET "https://api.tattoo.co/v1/studios?city=Los Angeles&state=CA" \
  -H "Authorization: Bearer YOUR_API_KEY"
```

```bash Filter by Rating
curl -X GET "https://api.tattoo.co/v1/studios?rating=4.5&verified=true" \
  -H "Authorization: Bearer YOUR_API_KEY"
```

```bash Geographic Search
curl -X GET "https://api.tattoo.co/v1/studios?latitude=34.0522&longitude=-118.2437&radius=10" \
  -H "Authorization: Bearer YOUR_API_KEY"
```

</CodeGroup>

## Step 5: Working with Artists

Now let's fetch artists from a specific studio:

```javascript Get Studio Artists
const studioId = "studio_123";
const response = await fetch(`https://api.tattoo.co/v1/artists?studio_id=${studioId}`, {
  headers: {
    'Authorization': 'Bearer YOUR_API_KEY',
    'Content-Type': 'application/json'
  }
});

const artists = await response.json();
```

## Step 6: Create a Booking

Here's how to create a new booking:

```javascript Create Booking
const bookingData = {
  artist_id: "artist_456",
  client_id: "client_789",
  appointment_date: "2025-02-01T14:00:00Z",
  duration_hours: 3,
  description: "Dragon sleeve tattoo",
  placement: "Right arm",
  estimated_price: 800
};

const response = await fetch('https://api.tattoo.co/v1/bookings', {
  method: 'POST',
  headers: {
    'Authorization': 'Bearer YOUR_API_KEY',
    'Content-Type': 'application/json'
  },
  body: JSON.stringify(bookingData)
});

const booking = await response.json();
```

## Common Parameters

Most endpoints support these common parameters:

| Parameter | Type | Description |
| --------- | ---- | ----------- |
| `page` | integer | Page number (default: 1) |
| `limit` | integer | Items per page (max: 100) |
| `search` | string | Search term |
| `sort` | string | Sort field |
| `order` | string | Sort order (asc/desc) |

## Error Handling

The API uses standard HTTP status codes and returns detailed error information:

```json Error Response
{
  "success": false,
  "error": {
    "message": "Validation failed",
    "code": "VALIDATION_ERROR",
    "details": [
      {
        "field": "email",
        "message": "Invalid email format"
      }
    ]
  },
  "meta": {
    "timestamp": "2025-01-15T10:00:00Z",
    "request_id": "req_123456789"
  }
}
```

## Rate Limits

Be aware of rate limits to avoid being throttled:

- **Authentication endpoints**: 5 requests per minute
- **General endpoints**: 100 requests per minute  
- **Search endpoints**: 50 requests per minute

<Tip>
Include the `X-RateLimit-*` headers in your monitoring to track usage and avoid hitting limits.
</Tip>
